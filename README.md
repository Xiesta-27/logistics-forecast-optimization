## 📋 项目背景与分析框架
### 业务背景
短途物流运输是电商履约的核心末端环节，直接决定用户履约体验与平台运营成本。
行业普遍存在三大核心痛点：
1.  货量预测偏差大，导致运力资源错配，高峰运力不足、平峰运力浪费
2.  车辆调度策略粗放，周转率、装载率低，运输成本高企
3.  应对预测波动的抗风险能力弱，方案稳定性差，易出现履约时效受损、成本超支的问题

本项目以物流业务数据为基础，核心解决 4 大业务问题：
1.  基于15天10分钟颗粒度的历史货量数据，预测未来1天各线路货量，并拆解到10分钟粒度；
2.  基于预测货量，以「自有车周转率最高、单车装载率最高、总成本最低」为目标，定位成本超支、运力浪费的核心根因，构建多目标运输调度优化模型；
3.  引入标准容器，在装卸效率与装载量之间做权衡，优化容器使用策略与调度方案，设计可落地的运力调度优化策略，量化验证策略的业务价值；
4.  分析预测误差带来的业务风险，平衡成本控制与履约稳定性

## 🧠 核心模型与思路

![大致思路图](./04-图表结果/论文核心图表/全文思路导图.png)

### 一：全链路指标体系搭建与数据底座建设
围绕物流揽收 - 运输 - 签收全环节，设计覆盖货量预测、车辆周转、装载效率、运输成本四大维度的北极星 + 过程监控指标体系；使用 Pandas 完成 15 天 10 分钟粒度历史货量数据清洗，结合 3σ 原则剔除异常值，通过时序可视化探查货量周期波动规律，为业务分析搭建标准化数据底座。

### 二：多维度业务拆解与核心痛点归因
针对运输成本超支、运力利用率低的问题，进行如下分析：
1.  **多维度下钻拆解**：从线路、时段、订单结构3个核心维度，对成本、周转率、装载率三大核心指标进行拆解，定位不同维度的指标缺口；
2.  **贡献度分析**：通过相关性分析与贡献度拆解，量化各因素对成本超支的影响，定位**货量预测偏差是成本超支的核心诱因**，贡献了68%的无效成本；
3.  **痛点深度分析**：原生LSTM预测模型对高峰段货量拟合不足，导致高峰运力缺口、平峰运力冗余，同时分段预测存在数据突变问题，进一步放大了预测偏差；
4.  **核心方案落地**：创新设计**带连续性约束的分段LSTM预测模型**，差异化拟合高峰/低谷段时序特征，添加时段平滑约束，最终将预测RMSE从原生LSTM模型的99.70降至34.21，从源头解决数据偏差问题。

### 三：多目标优化策略设计与效果验证
基于归因结论，将「周转率、装载率、成本最小化」三大冲突业务目标加权转化为单目标优化问题，构建 MILP 调度模型，设计动态运力调度、低货量线路串点合并、标准容器动态适配的组合策略；通过同质线路对照实验与假设检验验证策略显著性，最终实现自有车辆周转率提升至 85%，运输成本降低 23%，单车作业效率提升 42%。

### 四：业务风险识别与鲁棒性优化
针对现实中货量预测的不确定性，完成模型的抗扰动优化：
1.  扰动模拟：对预测货量加入±5%~±20%的随机扰动，模拟现实预测误差；
2.  灵敏度分析：量化预测误差对总成本、周转率、装载率的影响程度，识别模型敏感参数；
3.  动态优化：引入基于PPO算法的动态调整机制，在15%预测误差水平下，相比静态调度降低13%的成本超支，方案失效概率从38%降至12%。

## 💻 代码结构与运行说明
### 代码文件说明
| 文件名 | 核心功能 |
|--------|----------|
| q1_货量预测模型.py | 数据预处理、原生LSTM、分段LSTM、有约束分段LSTM模型实现与预测可视化 |
| q2_运输调度优化模型.py | 多目标MILP模型构建、任务生成、串点合并、车辆调度求解 |
| q3_标准容器调度优化.py | 容器使用决策、贪心+回溯算法实现、容器化调度优化 |
| q4_鲁棒性与灵敏度分析.py | 预测误差扰动模拟、灵敏度分析、鲁棒性评估、动态优化机制 |

- Python 3.9及以上版本
- 依赖安装：在代码目录下执行 `pip install -r requirements.txt` 即可一键安装所有依赖包
- 运行方式：按问题顺序依次运行对应py文件，即可复现论文所有结果与图表
